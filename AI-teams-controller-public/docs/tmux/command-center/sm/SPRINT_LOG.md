## How to Use This Log

### During Sprint
```bash
# When issue occurs, add row immediately:
| 08:30 | FE didn't follow TDD | SM | HIGH | OPEN | Boss flagged |
```

### At Retrospective
1. Read this log
2. Select REAL issues (not small/potential)
3. Create action items
4. Mark issues as RESOLVED

### Severity Guide
- **HIGH**: Caused sprint rejection, Boss flagged, major rework
- **MEDIUM**: Caused delay, multiple retests needed
- **LOW**: Minor inconvenience, quickly resolved
